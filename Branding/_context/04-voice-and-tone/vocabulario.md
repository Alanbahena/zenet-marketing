# Vocabulario

> Las palabras correctas no garantizan una buena pieza. Pero las palabras equivocadas garantizan una mala.

Este es el diccionario operativo de Zenet. Define el vocabulario que la marca usa, evita y reserva como propio. Junto con `voz-y-tono.md` y `reglas-de-redaccion.md`, completa el sistema de redacción de la marca.

Para los principios de voz que sostienen estas decisiones, ver `voz-y-tono.md`. Para la mecánica de la escritura (gramática, formato, español neutro, números), ver `reglas-de-redaccion.md`.

---

## 1. Cómo usar este vocabulario

Este documento se consulta — no se memoriza.

**Tres lentes principales:**

- **Vocabulario del operador (sección 2):** las palabras que Zenet usa porque son las palabras que el operador usa.
- **Vocabulario tech (sección 3):** las palabras que Zenet evita en comunicación con clientes, con sus reemplazos.
- **Vocabulario hype (sección 4):** las palabras que Zenet nunca usa, sin importar el contexto.

**Cuando dudes:**

1. Busca el término en el **glosario rápido** (sección 9) al final del documento.
2. Si no aparece ahí, revisa la sección temática correspondiente.
3. Si tampoco aparece, vuelve a `voz-y-tono.md` y aplica los nueve principios.

## 2. Vocabulario del operador (palabras que SÍ usamos)

Zenet escribe en el lenguaje de quien opera la cocina, no del equipo de TI ni del consultor genérico. Las palabras de esta sección son las que un operador reconoce sin esfuerzo — porque son las que ya usa cuando habla de su negocio.

### 2.1 Operación general

| Término | Cómo lo usamos |
|---|---|
| **Operación** | El conjunto del trabajo diario detrás del mostrador. *"Tu operación"* > *"tu negocio"* cuando hablamos de procesos. |
| **Sistema operativo** | Concepto central de la categoría. Zenet ES un sistema operativo (cognitivo) para back-of-house. |
| **Back-of-house** | La industria lo usa así. Aceptable en cualquier contexto. |
| **Trastienda** | Sinónimo más cálido de back-of-house. Útil en marketing emocional. |
| **Front-of-house / Sala** | La capa transaccional (POS, mesero, cliente). Zenet no opera aquí. |
| **Servicio** | El periodo activo de operación (turno, jornada). *"Antes del servicio..."*, *"durante el servicio..."*. |
| **Sucursal / sucursales** | La unidad operativa replicable. Zenet escala por sucursal. |
| **Cadena** | Operación con múltiples sucursales bajo una sola marca. |
| **Restaurante independiente** | El segmento principal de Zenet. Define el "no-cadena, no-franquicia". |
| **Crecimiento, escalamiento** | El estado del cliente objetivo: en expansión, planeando o ejecutando nuevas sucursales. |
| **Manual Operativo** | Producto/concepto Zenet — el manual vivo que generan los datos del sistema. Siempre con mayúsculas. |
| **Estandarización** | El primer paso de toda operación seria. Concepto central. |
| **Mise en place sistémico** | Concepto de marca: el "todo en su lugar" aplicado a la operación entera, no solo a la cocina. |

### 2.2 Cocina y recetas

| Término | Cómo lo usamos |
|---|---|
| **Receta / Receta estandarizada** | El bloque básico. *"Receta estandarizada"* cuando enfatizamos la disciplina. |
| **Escandallo** | Receta costeada (industry term). Preferir sobre "recipe costing". |
| **Ingrediente** | El componente de una receta. |
| **Insumo** | El producto comprable (puede o no ser ingrediente directo). Más amplio que "ingrediente". |
| **Mise en place** | En cursiva cuando se usa técnicamente (italics no estrictos en redes sociales). |
| **Brigada** | El equipo organizado de cocina. Concepto Escoffier que aún rige. |
| **Estación** | El puesto físico/funcional dentro de la cocina (estación de fríos, parrilla, postres). |
| **Preparación** | Lo que ocurre antes del servicio. |
| **Porción / Porcionado** | La unidad servida. Zenet trabaja con porciones, no con "platos" como unidad. |
| **Rendimiento** | El output esperado de un insumo o receta. *"Rendimiento de receta"*, *"rendimiento de un kilo de res"*. |
| **Merma** | La pérdida operativa (industry term). Siempre preferido sobre "desperdicio". |
| **Caducidad / Vencimiento** | Ambos válidos. "Caducidad" más formal, "vencimiento" más coloquial. |

### 2.3 Inventario y compras

| Término | Cómo lo usamos |
|---|---|
| **Inventario** | Concepto central. Plural: "los inventarios" cuando son múltiples sucursales. |
| **Insumo / Materia prima** | El producto que entra al inventario. |
| **Stock / Existencias** | Ambos válidos. "Stock" más industry, "existencias" más formal. |
| **Cámara** | Walk-in cooler / cámara fría. Industry term, preferido sobre "refrigerador grande". |
| **Proveedor** | Quien suministra los insumos. |
| **Orden de compra** | El documento/transacción de compra. |
| **Recepción de mercancía** | Cuando llega el insumo al restaurante. |
| **Conteo de inventario** | El acto de contar físicamente. |
| **Equivalencias / Conversión de unidades** | Concepto central de Zenet (kg ↔ g, taza ↔ ml). |
| **Unidad de medida (UM)** | La unidad en la que se cuenta y receta el insumo. |
| **Familia (de inventario)** | Grupo de insumos relacionados (carnes, lácteos, verduras). Concepto Zenet. |
| **Categoría (de receta)** | Grupo de recetas relacionadas (entradas, platos fuertes, bebidas). Concepto Zenet. |

### 2.4 Costos

| Término | Cómo lo usamos |
|---|---|
| **Costo** | Singular. Plural "costos" cuando se refiere a múltiples categorías. |
| **Costo teórico** | Lo que la receta estandarizada dice que debería costar. |
| **Costo real** | Lo que efectivamente costó al cierre del periodo. |
| **Desviación de costo** | La diferencia entre teórico y real. Concepto central de Zenet — donde aparece la interpretación. |
| **Margen** | Ganancia tras costos. Operadores entienden esto sin explicación. |
| **Food cost** | Industry term (% de costo de alimentos sobre venta). Aceptable en castellano. |
| **Punto de equilibrio** | Cuando ingresos cubren costos. |

### 2.5 Roles

Cuando se pueda, preferir términos basados en función sobre títulos formales (ver sección 8 — Lenguaje inclusivo).

| Término | Cómo lo usamos |
|---|---|
| **Dueño operador** | El owner-operator. Castellano-friendly y útil para nuestro segmento. |
| **Gerente operativo / Gerente de sucursal** | El responsable día a día de una sucursal. |
| **Chef ejecutivo** | El máximo responsable de cocina. |
| **Sous chef** | Mantener en francés, sin cursiva. Industry term universal. |
| **Chef de partida** | El responsable de una estación. |
| **Cocinero / cocinera** | Específico al rol de cocinar. |
| **Encargado de compras** | Quien gestiona insumos y proveedores. |
| **Operador** | Genérico cuando hablamos del usuario de Zenet. Cubre dueño, gerente, chef ejecutivo. |
| **Tu equipo / el personal** | Preferido sobre "tus empleados" o "tus colaboradores" cuando se puede. |

## 3. Vocabulario tech (NO usar con clientes)

Zenet evita el vocabulario técnico cuando habla con operadores, agencias, prensa o cualquier audiencia no-técnica. La regla: si la palabra requiere explicación tecnológica, no entra en el mensaje.

| Evitar | Reemplazar con |
|---|---|
| IA, inteligencia artificial | *(omitir cuando se pueda; describir el comportamiento)* — *"Zenet detecta..."*, *"Zenet sugiere..."* |
| Modelo, LLM, modelo de lenguaje | *"el sistema"*, *"Zenet"* |
| Agente, IA agéntica | *"Zenet"* directamente |
| RAG, embedding, vector store | (no mencionar) |
| Fine-tuning, training, modelo entrenado | (no mencionar) |
| Prompt | (no mencionar) |
| Token, inferencia | (no mencionar) |
| Machine learning, deep learning, ML | (no mencionar) |
| Algoritmo | *"lógica"*, *"criterio"*, o describir el comportamiento |
| Hallucination, alucinación | *"error"*, *"imprecisión"* |
| API, endpoint | *"conexión"* si aplica; usualmente no mencionar |
| Frontend, backend | *"la pantalla"* / *"lo que pasa detrás"* |
| Database, base de datos | *"donde se guarda la información"* |
| Stack tecnológico | (no mencionar) |
| Cloud, nube | *"en línea"* si aplica |
| UX, UI | *"experiencia de uso"* |
| Dashboard | *"tablero"*, *"panel"* |
| Onboarding técnico (auth, login flow) | *"el proceso de empezar"*, *"la configuración inicial"* |

**Excepción:** documentación técnica interna, conversaciones con desarrolladores, y artículos sobre arquitectura de Zenet pueden usar vocabulario tech sin restricción. La regla aplica a comunicación con cliente.

## 4. Vocabulario hype (NUNCA usar)

Estas palabras violan el principio "Sin hype" de la voz Zenet. **No tienen excepciones.** Si aparecen en una pieza, esa pieza está fuera de marca.

### 4.1 Superlativos prohibidos

- Increíble, asombroso, espectacular
- Revolucionario, disruptivo
- Único, exclusivo
- Líder, #1, el mejor, el más completo
- World-class, de clase mundial
- Garantizado, 100% garantizado
- Best-in-class, mejor-en-su-clase
- Next-gen, de próxima generación
- Imperdible
- Mágico, magia, mágicamente
- Wow, súper, mega
- Tremendo, brutal
- Insuperable, inigualable

### 4.2 Corporativismos prohibidos

- Sinergias
- Potencializar, potencia, potenciar (ojo: "amplificar" sí está bien)
- Soluciones integrales, soluciones a la medida
- Experiencia premium, experiencia única
- Empoderar, empoderamiento
- Rompiendo paradigmas
- Pensamiento out-of-the-box
- Game-changer, cambia-juego
- Stack tecnológico de punta
- Alineación estratégica
- Holístico, enfoque holístico
- Innovador (sobre-usado; preferir describir qué hace nuevo)
- Transformar (sobre-usado en B2B SaaS; preferir verbos más concretos cuando sea posible)

### 4.3 Frases de marketing barato

- *"¡Tap para descubrir!"*, *"¡Click para conocer más!"*
- *"¡No te lo pierdas!"*, *"¡Última oportunidad!"*
- *"¡Solo por tiempo limitado!"*
- *"¡Oferta exclusiva!"*, *"¡Promoción imperdible!"*
- *"Súmate a la revolución"*, *"Únete al cambio"*
- *"Lleva tu restaurante al siguiente nivel"*

**¿Cómo se reemplazan?** No se reemplazan — se reescribe la frase entera para que diga algo concreto. *"Lleva tu restaurante al siguiente nivel"* → *"Recupera 28 horas al mes en cada sucursal"*.

## 5. Términos de marca Zenet

### 5.1 Cómo se escribe "Zenet"

**Siempre así:** **Zenet**.
- Z mayúscula. Resto minúsculas.
- **Nunca:** ZENET, zenet, Z.E.N.E.T., Zen-net, Zen Net, zeNet.
- En contextos técnicos (URLs, package names, identificadores) `zenet` en minúsculas es aceptable porque es contexto distinto.
- No tiene punto al final cuando aparece como nombre.

### 5.2 Cómo nombramos al "agente IA"

**Decisión de marca: el producto es el agente.** No hay nombre separado para la IA.

| ✅ Sí | ❌ No |
|---|---|
| *"Zenet detectó una desviación..."* | *"Zenet AI detectó..."* |
| *"Zenet te sugiere revisar..."* | *"El bot de Zenet sugiere..."* |
| *"Zenet revisa tu inventario todos los días."* | *"ZenAI revisa tu inventario..."* |
| *"Lo que Zenet ve cuando analiza tus datos..."* | *"Lo que el asistente de Zenet ve..."* |

**Por qué:** simplifica la marca y evita la trampa de "dos productos en uno". El agente no es algo aparte de Zenet. Es como Zenet se expresa.

**Excepción técnica/interna:** en documentación de desarrollo, presentaciones de arquitectura, o conversaciones con stakeholders técnicos, *"el agente"* o *"el sistema agéntico"* son válidos.

### 5.3 Secciones del producto

Estos son los nombres canónicos de las secciones del producto Zenet. Siempre con mayúscula inicial cuando se refiere a la sección del software.

- **Bienvenida** — la entrada al producto.
- **Clasificación** — la fase de definir el tipo de restaurante.
- **Configuración** — unidades, familias, categorías.
- **Alineamiento** — vincular recetas con inventario.
- **Estructura** — enriquecer el inventario.
- **Normalización** — resolver inconsistencias entre unidades.
- **Manual Operativo** — el output final, el manual vivo.

Cuando se mencionan en plural o en concepto general, se pueden usar sin mayúscula: *"las secciones de estandarización"*, *"el flujo de configuración inicial"*.

### 5.4 Conceptos propios

| Término | Cómo se usa |
|---|---|
| **Manual Operativo** | Siempre con M y O mayúsculas cuando se refiere al artefacto producido por Zenet. |
| **Mise en place sistémico** | Concepto de marca; minúsculas (excepto al inicio de oración o título). |
| **Bloques operativos** | Minúsculas. Concepto de la visión a futuro. |
| **Sistema operativo cognitivo** | Concepto de la categoría. Minúsculas en uso descriptivo. |

## 6. Decisiones de traducción

Zenet opera primariamente en español. Pero algunos términos extranjeros (inglés industria, francés cocina) son más reconocibles que sus traducciones. Esta sección define cuáles mantenemos y cuáles traducimos.

### 6.1 Términos extranjeros que MANTENEMOS

| Término | Razón |
|---|---|
| *Mise en place* | Universal en industria gastronómica. Mantener en francés (cursiva en uso técnico). |
| *Sous chef* | Mismo razonamiento. Sin cursiva en uso normal — ya integrado. |
| *Brigada* | Versión castellana del concepto Escoffier. Aceptable. |
| *Food cost* | Industry-acceptable. Más usado que "porcentaje de costo de alimentos". |
| *Back-of-house / Front-of-house* | Industry-aceptable. Coexisten con "trastienda" y "sala". |
| *Onboarding* | Más usado que "incorporación". Aceptable en marketing y producto. |
| *POS* | Más reconocido que "punto de venta" en industria. Coexisten. |
| *Demo* | Aceptable. |

### 6.2 Términos extranjeros que TRADUCIMOS

| Inglés | Español Zenet |
|---|---|
| Inventory | Inventario |
| Recipe | Receta |
| Stock | Existencias o stock (ambos OK; "existencias" más formal) |
| Click / clicking | Clic / hacer clic (no "click") |
| Email | Correo o email (ambos OK) |
| App | Aplicación o app (ambos OK; "app" más casual) |
| Item | Artículo o ítem (ambos OK; "artículo" más castizo) |
| User | Usuario (en marketing preferir "operador" o "cliente") |
| Customer | Cliente |
| Pricing | Precios |
| Feature | Función o funcionalidad (no "feature" en cliente) |
| Bug | Error |
| Update | Actualización |
| Sign in / Login | Iniciar sesión |
| Sign up | Registrarse |
| Support | Soporte |
| Dashboard | Tablero o panel |

### 6.3 Reglas de cursiva para extranjerismos

- *Cursiva* cuando se usa el término por primera vez en un texto largo (artículo, post largo).
- Sin cursiva cuando ya es palabra integrada al castellano de uso común (ej. *sous chef* en uso normal).
- En interfaces de producto, sin cursiva (las interfaces no permiten formato consistente).

## 7. Pronombres y trato

### 7.1 Tratamiento por defecto: "tú"

| Superficie | Tratamiento |
|---|---|
| Producto (todas las pantallas) | Tú |
| Marketing (landing, redes, email) | Tú |
| LinkedIn (fundador y marca) | Tú |
| Anuncios pagados | Tú |
| Soporte 1-a-1 | Tú (a menos que el cliente pida usted) |

### 7.2 Cuándo cambiar a "usted"

- **Documentos legales** (Términos y Condiciones, Política de Privacidad, contratos).
- **Press releases** y comunicación con medios formales.
- **Soporte 1-a-1 cuando el cliente lo pide explícitamente o lo señala con su propio uso.**

### 7.3 Plural

Siempre **"ustedes"**. Nunca "vosotros" (Zenet opera en LATAM, no España).

### 7.4 Regla de consistencia

**Nunca mezclar tú y usted en un mismo texto.** Si una pieza empieza con tú, todo el texto es con tú — sin excepciones, sin partes que se sientan más formales.

## 8. Lenguaje inclusivo

### 8.1 La regla principal: rol antes que género

Cuando se pueda, usar lenguaje basado en función que evite gendering. Esto no es decoración — es escritura más clara.

| ✅ Preferido | ❌ Evitar |
|---|---|
| Tu equipo | Tus empleados / tus colaboradores |
| El personal | Los empleados |
| Quien opera la cocina | El dueño o la dueña |
| Quien lleva las compras | El encargado o la encargada de compras |
| El restaurante | El dueño o la dueña |

### 8.2 Cuando hay que generizar: masculino genérico

Cuando no hay alternativa basada en rol, usar masculino genérico.

- "Los operadores", "los chefs", "los dueños" cuando hablamos del segmento.
- "El operador" como genérico (no "el/la operador/a").

### 8.3 Lo que Zenet NO usa

| ❌ Forma que no usamos | Por qué |
|---|---|
| L@s, lxs (l@s operador@s, lxs chefxs) | Crea fricción al leer; performativo. |
| "Todes" en lugar de "todos" | Performativo en el segmento. |
| Doble forma forzada (los operadores y las operadoras) | Sobrecarga el texto excepto en momentos donde la inclusión es el punto. |

### 8.4 En comunicación 1-a-1

Adaptar al individuo. Si la persona es Carla, escribir en femenino. Si la persona es Diego, en masculino. Si no se sabe, masculino genérico hasta tener señal.

## Notas

- **Origen.** Este vocabulario sintetiza decisiones tomadas en `voz-y-tono.md` (los nueve principios) y `personalidad-y-arquetipo.md` (la postura del Sabio + Cuidador) en palabras concretas. El glosario alfabético se construyó priorizando los términos donde la confusión es más probable.
- **Documento vivo.** El vocabulario crece con el tiempo. Reglas para sumar términos nuevos:
  - Si una palabra surge en comunicación con cliente y genera duda — se documenta.
  - Si un nuevo módulo del producto introduce un concepto — se documenta.
  - Si un competidor o tendencia introduce un término que Zenet debe adoptar o evitar — se documenta.
- **Idioma base.** Español neutro latinoamericano con anclaje mexicano por el segmento inicial. Cuando Zenet expanda a otros mercados de LATAM (Colombia, Argentina, Chile), las decisiones específicas regionales se anexarán como secciones por país, no se reemplazará el documento base.
- **Uso por agentes IA / freelancers / equipo interno.** Este documento es referencia obligatoria. Cualquier copia que se redacte para Zenet debe pasar por las decisiones aquí establecidas antes de publicarse.
- **Documentos hermanos.** `voz-y-tono.md` (los principios), `reglas-de-redaccion.md` (la mecánica), y este documento (las palabras) forman el sistema voice-and-tone completo.
